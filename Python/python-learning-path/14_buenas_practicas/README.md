# 14 - Buenas Prácticas en Python

## Concepto
Las buenas prácticas ayudan a escribir código claro, eficiente y fácil de mantener.

## Analogía sencilla
Piensa en escribir una receta: debe ser clara para que cualquiera la entienda y siga.

## Principales buenas prácticas
- Seguir PEP8 (guía de estilo de Python)
- Nombres descriptivos para variables y funciones
- Estructura de proyectos ordenada

## Ejemplo explicado
```python
# Ejemplo de nombres descriptivos
def calcular_promedio(lista_numeros):
    return sum(lista_numeros) / len(lista_numeros)

# Estructura de proyecto
# proyecto/
# ├── main.py
# ├── utils.py
# └── datos/
```

## Errores comunes
- Nombres poco claros
- Código sin comentarios
- Mezclar lógica en un solo archivo

## Buenas prácticas
- Comentar el código
- Separar funciones y módulos
- Usar espacios y sangría correcta

## Ejercicios prácticos
1. Refactoriza un código usando nombres claros.
2. Organiza un proyecto en carpetas y archivos.

## Mini reto
Revisa un script y mejora su legibilidad siguiendo PEP8.
